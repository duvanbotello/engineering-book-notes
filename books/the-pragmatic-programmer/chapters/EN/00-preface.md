# 00 - Preface (first edition)

> Quick navigation: [Core idea](#core-idea) · [Key concepts](#key-concepts) · [Actions](#actions) · [Related notes](#related-notes)

## Core idea

There is no point in developing software if there is no real commitment to doing it well. The starting point is not technology, but professional attitude toward quality.

## Key concepts

- **Quality as intention:** software that "works" is not enough if we ignore how it is built.
- **Ownership from the start:** the way we work defines long-term outcomes.
- **Professional judgment:** urgency does not justify neglecting fundamentals.

## Quotes (short only)

- "There is no point in developing software if we do not care about doing it well."

## My interpretation

The preface works as a direct warning: building software without judgment becomes costly, both technically and humanly. Doing it well is not perfectionism; it is responsibility.

## Practical lessons

- Defining a quality baseline before starting avoids late-stage arguments.
- Today's small decisions become tomorrow's debt or stability.

## Questions

- What does "doing it well" mean in my current team context?
- Which compromises are we normalizing that will slow us down later?

## Actions

- [ ] Align on minimum quality criteria for new deliveries.
- [ ] In each delivery, check whether we met intent, not only functionality.
