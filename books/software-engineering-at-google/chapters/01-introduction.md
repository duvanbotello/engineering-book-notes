# 01 - Introduction

## Core idea

Software engineering is programming integrated with time, scale, and human collaboration. The chapter frames engineering quality as a long-term outcome of both code and organizational habits.

## Key concepts

- Programming vs. software engineering
- Tradeoffs across time and scale
- Socio-technical systems and maintainability

## Quotes (short only)

- "Software engineering is programming integrated over time."

## My interpretation

The key message is that correctness today is not enough. A useful solution must remain understandable, adaptable, and teachable as systems and teams evolve.

## Practical lessons

- Evaluate decisions by maintenance cost, not only implementation speed.
- Capture rationale behind changes to reduce future ambiguity.
- Treat team practices as part of technical architecture.

## Questions

- Which current team decisions optimize short-term output but hurt long-term adaptability?
- Where should we document tradeoffs so new contributors can understand context quickly?
